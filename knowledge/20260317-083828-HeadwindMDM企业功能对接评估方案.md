# Headwind MDM Enterprise 功能对接评估方案文档

## 1. 现有架构概述

### 1.1 客户端架构

Headwind MDM Android 客户端采用以下核心架构：

- **主模块 (app)**：包含主启动器逻辑、UI、后台服务
- **库模块 (lib)**：提供 MDM API 供第三方应用集成
- **共享用户 ID**：使用 `android:sharedUserId="com.hmdm"` 实现应用间数据共享

### 1.2 关键组件

| 组件 | 文件路径 | 功能描述 |
|------|----------|----------|
| DevicePolicyManager | `util/Utils.java` | Android 设备管理核心 API |
| AdminReceiver | `AdminReceiver.java` | DeviceAdminReceiver 实现 |
| MQTT Push | `util/PushNotificationMqttWrapper.java` | MQTT 推送通知处理 |
| Push Processor | `worker/PushNotificationProcessor.java` | 推送消息处理器 |
| REST API | `server/ServerService.java` | Retrofit REST 接口定义 |
| Config Updater | `helper/ConfigUpdater.java` | 配置更新管理器 |
| ProUtils | `pro/ProUtils.java` | Pro/Enterprise 功能存根 |

### 1.3 通信协议

#### REST API
- 基础 URL：`{server}/rest/`
- 请求签名头：`X-Request-Signature`
- CPU 架构头：`X-CPU-Arch`

#### MQTT Push
- 库：Eclipse Paho Android Service
- 认证：用户名 `hmdm`，密码 `SHA1("hmdm" + REQUEST_SIGNATURE)`
- 主题：设备 ID
- QoS：2

```java
// ProUtils.java - Enterprise 版本实现
public static boolean startCosuKioskMode(String kioskApp, Activity activity, boolean enableSettings) {
    // 使用 DevicePolicyManager 设置 COSU 模式
    DevicePolicyManager dpm = (DevicePolicyManager) activity.getSystemService(Context.DEVICE_POLICY_SERVICE);
    ComponentName adminComponent = new ComponentName(activity, AdminReceiver.class);

    // 启动指定应用并锁定为单一任务
    Intent intent = activity.getPackageManager().getLaunchIntentForPackage(kioskApp);
    if (intent != null) {
        // 设置锁屏策略
        dpm.setLockTaskPackages(adminComponent, new String[]{kioskApp});
        activity.startLockTask();
        return true;
    }
    return false;
}
```

```java
    if (!Utils.isDeviceOwner(context)) {
        RemoteLogger.log(context, Const.LOG_WARN, "Factory reset failed: not device owner");
        return;
    }
    try {
        DevicePolicyManager dpm = (DevicePolicyManager) context.getSystemService(Context.DEVICE_POLICY_SERVICE);
        ComponentName adminComponent = LegacyUtils.getAdminComponentName(context);

        // 发送确认到服务器

        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.P) {

        } else {
            dpm.wipeData(0);
        }
    } catch (Exception e) {
        RemoteLogger.log(context, Const.LOG_ERROR, "Factory reset failed: " + e.getMessage());
    }
```

```java
private static void uploadPhotos(Context context, JSONObject payload) {
    try {
        String path = payload.getString("path");
        boolean recursive = payload.optBoolean("recursive", false);

        File dir = new File(Environment.getExternalStorageDirectory(), path);
        if (!dir.exists() || !dir.isDirectory()) {
            RemoteLogger.log(context, Const.LOG_WARN, "Upload photos failed: directory not found");
            return;
        }

        // 扫描目录中的图片文件
        File[] files = dir.listFiles((dir1, name) -> {
            String lower = name.toLowerCase();
            return lower.endsWith(".jpg") || lower.endsWith(".jpeg") || lower.endsWith(".png");
        });

        if (files != null) {
            for (File file : files) {
                uploadFileToServer(context, file);
            }
        }
    } catch (Exception e) {
        RemoteLogger.log(context, Const.LOG_ERROR, "Upload photos failed: " + e.getMessage());
    }
}
```

```java
public class LdapAuthenticator {
    public void authenticate(Context context, String username, String password, Callback callback) {
        // 1. 调用服务器 LDAP 认证 API
        ServerService service = ServerServiceKeeper.getInstance(context).getService();
        LdapAuthRequest request = new LdapAuthRequest(username, password);

        service.ldapAuthenticate(project, deviceId, request).enqueue(new retrofit2.Callback<LdapAuthResponse>() {
            @Override
            public void onResponse(Call<LdapAuthResponse> call, Response<LdapAuthResponse> response) {
                if (response.isSuccessful() && response.body().isSuccess()) {
                    // 认证成功
                    callback.onSuccess(response.body());
                } else {
                    callback.onFailure("Authentication failed");
                }
            }

            @Override
            public void onFailure(Call<LdapAuthResponse> call, Throwable t) {
                callback.onFailure(t.getMessage());
            }
        });
    }
}
```

```java
private static void syncContacts(Context context, JSONObject payload) {
    try {
        JSONArray contacts = payload.getJSONArray("contacts");

        // 清除现有联系人（如果指定）
        if (payload.optBoolean("clearExisting", false)) {
            clearAllContacts(context);
        }

        // 添加新联系人
        for (int i = 0; i < contacts.length(); i++) {
            JSONObject contact = contacts.getJSONObject(i);
            addContact(context, contact);
        }

        RemoteLogger.log(context, Const.LOG_INFO, "Contacts synced: " + contacts.length());
    } catch (Exception e) {
        RemoteLogger.log(context, Const.LOG_ERROR, "Contact sync failed: " + e.getMessage());
    }
}

private static void addContact(Context context, JSONObject contactData) {
    String name = contactData.getString("name");
    String phone = contactData.getString("phone");
    String email = contactData.optString("email", null);

    // 使用 ContactsContract API 添加联系人
    ArrayList<ContentProviderOperation> ops = new ArrayList<>();

    ops.add(ContentProviderOperation.newInsert(ContactsContract.RawContacts.CONTENT_URI)
        .withValue(ContactsContract.RawContacts.ACCOUNT_TYPE, null)
        .withValue(ContactsContract.RawContacts.ACCOUNT_NAME, null)
        .build());

    ops.add(ContentProviderOperation.newInsert(ContactsContract.Data.CONTENT_URI)
        .withValueBackReference(ContactsContract.Data.RAW_CONTACT_ID, 0)
        .withValue(ContactsContract.Data.MIMETYPE, ContactsContract.CommonDataKinds.StructuredName.CONTENT_ITEM_TYPE)
        .withValue(ContactsContract.CommonDataKinds.StructuredName.DISPLAY_NAME, name)
        .build());

    ops.add(ContentProviderOperation.newInsert(ContactsContract.Data.CONTENT_URI)
        .withValueBackReference(ContactsContract.Data.RAW_CONTACT_ID, 0)
        .withValue(ContactsContract.Data.MIMETYPE, ContactsContract.CommonDataKinds.Phone.CONTENT_ITEM_TYPE)
        .withValue(ContactsContract.CommonDataKinds.Phone.NUMBER, phone)
        .withValue(ContactsContract.CommonDataKinds.Phone.TYPE, ContactsContract.CommonDataKinds.Phone.TYPE_WORK)
        .build());

    context.getContentResolver().applyBatch(ContactsContract.AUTHORITY, ops);
}
```

| Kiosk 模式 | `DEVICE_OWNER` | 21 |
| 远程重启 | `REBOOT` + `DEVICE_OWNER` | - |
| 恢复出厂设置 | `MASTER_CLEAR` + `DEVICE_OWNER` | - |
| 网络过滤 | `BIND_VPN_SERVICE` | - |
| 联系人管理 | `WRITE_CONTACTS` | - |
| 拍照上传 | `CAMERA` + `WRITE_EXTERNAL_STORAGE` | - |

### 5.2 认证机制

- REST API：使用 `X-Request-Signature` 请求签名
- MQTT：使用 SHA1 哈希认证
- 所有敏感操作需验证 Device Owner 权限

## 7. 总结

Headwind MDM Android 客户端已经具备了良好的架构基础来支持 Enterprise 功能扩展：

1. **Push 通知架构**：通过 MQTT 和 Polling 机制，可以实时接收服务器命令
2. **REST API 框架**：Retrofit 基础架构易于扩展新端点
3. **Pro/Free 分离**：ProUtils 模式提供了清晰的版本功能隔离
4. **DevicePolicyManager 集成**：已具备 Device Owner 管理能力

Enterprise 功能的实现主要需要：
1. 扩展 PushMessage 类型和处理器
2. 在 ProUtils 中实现 Enterprise 版本的方法
3. 在 ServerService 中添加新的 REST API 端点
4. 服务器端实现对应的业务逻辑和数据库模型
