# Headwind MDM 端到端(E2E)测试文档

## 概述

本文档用于在真机上模拟测试Headwind MDM的10个企业功能。每个功能包含：
- 功能说明
- 前后端实现概览
- 测试步骤
- 预期结果
- 验证方法

---

## 测试环境要求

### 硬件
- Android设备（Android 6.0+推荐）
- USB连接或网络连接

### 软件
- 编译好的Headwind MDM APK
- 服务器端已部署并运行
- ADB工具（用于日志抓取）

### 服务器
- 已部署的HMDM服务器
- 服务器URL已知（如 http://192.168.1.100:8080）

## 功能1: 位置跟踪 (Location Tracking)

### 功能说明
设备定期上传GPS位置到服务器，管理员可以在Web控制台查看设备位置。

### 后端实现
- **API端点**: `PUT /plugins/devicelocations/public/update/{number}`
- **文件**: `DeviceLocationResource.java`

### 前端实现
- **Worker**: `LocationUploadWorker.java`
- **Service**: `LocationService.java`

### 测试步骤

1. **准备服务器**
   ```bash
   # 确保服务器已启动
   # 确认数据库已创建相关表
   ```

2. **配置设备**
   - 安装并启动HMDM应用
   - 完成设备注册（输入服务器URL和设备ID）
   - 在设置中启用"位置跟踪"

3. **触发位置上传**
   - 设备会周期性自动上传
   - 或手动触发：等待WorkManager定时任务执行

4. **验证结果**
   - 访问服务器API验证数据：
   ```bash
   curl -X GET http://服务器地址/plugins/devicelocations/public/latest/设备编号
   ```
   - 或检查数据库`device_locations`表

### 预期结果
- API返回JSON包含经纬度、时间戳
- 数据库成功插入位置记录

## 功能2: 照片上传 (Photo Upload)

### 功能说明

3. **验证结果**
   ```bash
   # 检查照片列表API
   ```

- 照片元数据保存到数据库

## 功能3: 远程控制 (Remote Control)

### 功能说明
管理员可以远程查看设备屏幕、截屏、锁屏、重启设备。

### 后端实现
- **API端点**:
  - `GET /plugins/devicecontrol/session/{number}` - 获取会话
  - `POST /plugins/devicecontrol/signal/{number}` - 发送控制信号
- **文件**: `DeviceControlResource.java`

### 前端实现
- **Service**: `RemoteControlService.java`
- **处理的信号**: screenshot, lock, reboot, factory_reset

### 测试步骤

1. **启动远程控制服务**
   ```bash
   adb shell am startservice -n com.hmdm.launcher/.service.RemoteControlService
   ```

2. **发送远程控制信号**
   ```bash
   # 截屏
   curl -X POST "http://服务器地址/plugins/devicecontrol/signal/设备编号" \
     -H "Content-Type: application/json" \
     -d '{"type":"screenshot"}'

   # 锁屏
   curl -X POST "http://服务器地址/plugins/devicecontrol/signal/设备编号" \
     -H "Content-Type: application/json" \
     -d '{"type":"lock"}'

   # 重启
   curl -X POST "http://服务器地址/plugins/devicecontrol/signal/设备编号" \
     -H "Content-Type: application/json" \
     -d '{"type":"reboot"}'
   ```

3. **检查设备日志**
   ```bash
   adb logcat -d | grep -i "remote"
   ```

### 预期结果
- 设备收到推送消息
- 执行相应的操作（截屏、锁屏等）
- 日志显示操作执行记录

## 功能4: 网络过滤 (Network Filter)

### 功能说明
通过VPN方式拦截设备网络流量，根据规则过滤或记录网络请求。

### 后端实现
- **API端点**:
  - `GET /plugins/networkfilter/rules/{number}` - 获取规则
  - `POST /plugins/networkfilter/logs/{number}` - 上传日志
  - `GET /plugins/networkfilter/stats/{number}` - 流量统计
- **文件**: `NetworkFilterResource.java`

### 前端实现
- **Service**: `NetworkFilterService.java` (基于Android VpnService)

### 测试步骤

1. **请求VPN权限**
   - 首次启动时系统会弹出VPN权限申请
   - 同意授权

2. **启动网络过滤**
   ```bash
   adb shell am startservice -n com.hmdm.launcher/.service.NetworkFilterService
   ```

3. **获取过滤规则**
   ```bash
   curl -X GET "http://服务器地址/plugins/networkfilter/rules/设备编号"
   ```

4. **检查流量统计**
   ```bash
   curl -X GET "http://服务器地址/plugins/networkfilter/stats/设备编号"
   ```

5. **停止服务**
   ```bash
   adb shell am stopservice -n com.hmdm.launcher/.service.NetworkFilterService
   ```

### 预期结果
- VPN服务成功启动
- 前台显示通知
- API返回过滤规则（如果有配置）

## 功能5: 联系人同步 (Contact Sync)

### 功能说明
双向同步设备联系人：设备→服务器、服务器→设备。

### 后端实现
- **API端点**:
  - `GET /plugins/contacts/{number}` - 获取联系人
  - `PUT /plugins/contacts/{number}` - 上传联系人
  - `DELETE /plugins/contacts/{number}` - 删除联系人
- **文件**: `ContactsSyncResource.java`

### 前端实现
- **Worker**: `ContactSyncWorker.java`

### 测试步骤

1. **上传联系人**
   ```bash
   # 准备联系人JSON
   cat contacts.json
   # {"contacts":[{"name":"Test User","phone":"1234567890","email":"test@example.com"}]}

   curl -X PUT "http://服务器地址/plugins/contacts/设备编号" \
     -H "Content-Type: application/json" \
     -d @contacts.json
   ```

2. **获取联系人**
   ```bash
   curl -X GET "http://服务器地址/plugins/contacts/设备编号"
   ```

3. **搜索联系人**
   ```bash
   curl -X GET "http://服务器地址/plugins/contacts/search/设备编号?query=Test"
   ```

4. **删除联系人**
   ```bash
   curl -X DELETE "http://服务器地址/plugins/contacts/设备编号"
   ```

### 预期结果
- 上传后返回保存成功消息
- 获取返回联系人JSON数组
- 搜索返回匹配结果
- 删除后联系人被清除

## 功能6: Kiosk模式 (Kiosk Mode)

### 功能说明
锁定设备到指定应用，限制用户只能使用特定应用。

### 后端实现
- 配置在`ServerConfig`中通过`kioskPackage`字段下发

### 前端实现
- **Utils**: `ProUtils.java` - `lockToApp()`, `unlockFromApp()`
- **Service**: `CheckForegroundAppAccessibilityService.java`

### 测试步骤

1. **配置Kiosk应用**
   - 在服务器配置中设置Kiosk包名
   - 设备同步配置

2. **启用Kiosk模式**
   ```bash
   # 通过设备端触发
   adb shell am broadcast -a com.hmdm.launcher.action.LOCK_TO_APP \
     --es package "com.android.settings"
   ```

3. **检查设备状态**
   - 设备应被锁定在指定应用
   - 按Home键无响应

4. **解锁**
   ```bash
   adb shell am broadcast -a com.hmdm.launcher.action.UNLOCK_FROM_APP
   ```

### 预期结果
- 设备被锁定到指定应用
- 用户无法退出该应用
- 解锁后恢复正常

2. **检查设备日志**
   ```bash
   adb logcat -d | grep -i "reboot"
   ```

### 预期结果

## 功能9: 恢复出厂设置 (Factory Reset)

### 功能说明
管理员远程执行恢复出厂设置（危险操作）。

### 后端实现
- 通过`DeviceControlResource`发送`factory_reset`信号

### 前端实现
- `PushNotificationProcessor`处理`REMOTE_FACTORY_RESET`消息

### ⚠️ 警告
此操作会清除设备所有数据！测试时谨慎使用。

### 测试步骤

1. **发送恢复出厂设置命令**
   ```bash
   curl -X POST "http://服务器地址/plugins/devicecontrol/signal/设备编号" \
     -H "Content-Type: application/json" \
     -d '{"type":"factory_reset"}'
   ```

2. **检查设备**
   - 设备应开始恢复出厂设置流程

### 预期结果
- 设备显示恢复出厂设置确认界面
- 确认后执行恢复
